RBD

See Section Error: Reference source not found

 They are well suited for systems with non-repairable components following independent behaviour. The methods RBD and FTA should be used for system reliability analysis if the system can be represented as a serial or parallel arrangement or a combination of both without dependencies. Repair, sequences, re-configuration and dependencies cannot be modelled. The calculation rules are identical for RBD and FTA, just the graphical presentation of the system behaviour is different.

Inputs: System architecture, component or sub-system failure rates/reliabilities

Output: System Reliability

Event Tree Analysis

See Section Error: Reference source not found

 Analyse the sequence of events following an initiating event Only one initiating event can be analysed in each analysis. Difficult to analyze common cause failures.

Input: List of initiating event, additional event and barriers/mitigations means. Probabilities of events

Output: Probability of certain consequences depending on initiating event

Markov Process modelling

See Section Error: Reference source not found

 State space based, Markov should be used to analyse repairable system, reconfigurations and dynamic behaviour. Markov uses constant failure rates/repair rates. The model size increases rapidly with number of components, for components, there are system states.

Input: System architecture

Dynamic system behaviour

Transition rates

Output: State probability

Petri network

See Section Error: Reference source not found

 Dynamic behaviour can be analysed. Deterministic and stochastic transitions. Petri nets should be used to model the various states of the system and how it evolves from one state to another when events occur. Not limited to constant transition rates and transition diagram might be easier to model compared to Markov as less subject to combinatory explosion of states.

Inputs: System architecture

Dynamic system behaviour

Transitions (deterministic, arbitrary distribution)

Output: Probability of certain states/transitions

Bayesian networks

See Section Error: Reference source not found

 Directed acyclic graphs represent explicitly dependencies, status of child node depending on status of parent nodes. Allows modelling of dependencies. Allows merging knowledge of diverse natures (in-service data, expert knowledge etc.). Allows modelling of Common Cause Failure through explicit modelling of dependencies.

Input: Set of random variables and causal relationships between them expressed as conditional probability table

Output: Probability of certain random variables
